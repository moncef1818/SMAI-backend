Implementation notes for backend developer
TopicRequirement
Event routingRoute by LogSource field in EventEnvelope. Never parse filename. Values: ProcessMonitor | FileMonitor | NetworkMonitor | AuthMonitor | UsbMonitor | RegistryMonitor
Hard vs scoredHard rules: evaluate synchronously per event, emit alert immediately. Scored rules (D-series only): accumulate per device event, emit one alert with total score and all matching rule IDs after Remove event.
Null safetyAll derived fields (IatCoefficientOfVariation, ExfiltrationZScore, FailCount30s, etc.) can be null — always null-check before comparison. A null value means "not enough data yet", never fire the rule on null.
Backend stateOnly A-4c requires backend sliding window state (PID→timestamps). All other rules are stateless evaluations of pre-computed agent fields.
DeduplicationSuppress duplicate alerts: do not re-fire the same rule for the same host_id within the cooldown: A-series=5min, B-series=10min, C-1a=30s, C-1b=5min, D-series per device_id.
Alert payloadEvery alert must include: rule_id, severity, host_id, log_source, timestamp, triggering_field_values (the exact fields and values that matched), mitre_technique, raw_event_payload (the full original JSON).
IsSigned=nullNever treat null as false. IsSigned null = "check not performed". IsSigned false = "checked and unsigned". These are different. Only fire A-1b on IsSigned=false (explicit).
